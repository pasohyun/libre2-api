## 3) 메인 대시보드 구현 현황

### 3.1 구현 완료
- 채널별 판매가 추이 차트
  - 일별/월별 토글 지원
  - `판매처 x 채널`(네이버/쿠팡) 최대 10개 라인 분리 시각화
  - 라인별 ON/OFF 토글 지원(네이버 5개/쿠팡 5개 그룹)
  - 데이터 소스: 판매처 개별 타임라인(`mall/timeline`) 합성
- 기준가 이하 판매처 현황 카드
  - 전체/네이버/쿠팡/기타 채널별 카운트 표시
  - 클릭 시 채널 필터 즉시 반영
- 기준가 이하 판매처 테이블
  - 컬럼: 선택, 채널, 판매처, 상품명, 판매가, 단가, 링크, 확인시간, 사이트화면
  - 페이지네이션(이전/다음/페이지 번호) 적용
  - 체크박스 선택 후 일괄 삭제 지원(실제 DB 삭제 반영)
- 수동확인(수량 확정) 플로우
  - `확인필요`, `가격역산(보정)`, `텍스트분석(범위초과)` 항목에 대해 수동 수량 확정 가능
  - 확정 후 단가 재계산 및 DB 업데이트
- HTML 카드 미리보기 및 이미지 생성 트리거
  - HTML 카드 모달 제공
  - 필요 시 단건 카드 생성 API 호출 가능
- 크롤링 제어
  - 대시보드에서 수동 크롤링 실행
  - 실행 상태(진행 여부/최근 시작/종료/오류) 확인

### 3.2 부분 구현(MVP/폴백)
- 일부 차트/셀러 데이터는 API 데이터 부재 시 샘플 데이터로 폴백
- 마켓 필터는 MVP 형태로 단순 매칭(판매처명 기반) 처리

### 3.3 미구현 또는 제한사항
- 채널 칩 중 비활성 채널은 UI에 "준비중"으로 표시
- 일부 고급 분석 로직(정교한 마켓 분류, 경보 자동화)은 확장 여지 존재

## 4) 네이버 채널 구현 현황

### 4.1 구현 완료 (수집/정제)
- 네이버 쇼핑 Open API 연동 및 페이징 수집
  - 최대 1000건 범위 수집 루프
  - 재시도/타임아웃/요청 간 딜레이 처리
- 리브레2 대상 상품 필터링
  - 비대상 CGM(덱스콤/가디언 등) 제외
  - 액세서리/부속품 노이즈 제외
  - 카테고리 기반 정합성 필터 적용
- 단가 계산 로직
  - 상품명 기반 수량 추출(정규식)
  - 가격 역산 보정
  - `calc_method`, `calc_valid` 관리
- 판매처명 표준화
  - 예: `글루어트/글루코핏/글핏몰 -> 랜식(글핏몰)`
  - `닥다몰 -> 닥터다이어리(닥다몰)`, `무화당 -> 닥터다이어리(무화당)`으로 분리 관리
- DB 저장
  - `snapshot_id`, `snapshot_at` 단위 저장으로 실행 단위 데이터 구분

### 4.2 구현 완료 (네이버 채널 화면)
- 네이버 채널 "주요 셀러" 화면
  - 주요 셀러 카드(현재 단가, 7일 변동폭, 기준가 이하 횟수) 제공
  - 고정 5개 주요 셀러 순서 고정:
    - `닥터다이어리(닥다몰)`, `랜식(글핏몰)`, `레디투힐`, `메디프라`, `필라이즈`
  - 기타 네이버 판매처 병행 노출(가격 하락순)
- 네이버 채널 추이 그래프
  - 고정 5개 주요 셀러 라인만 표시
  - 데이터 소스: 5개 판매처 개별 타임라인(`mall/timeline`) 합성
- 네이버 판매처 상세 타임라인
  - 판매처별 시점 단위 최저가 히스토리 제공
  - 확인시간/판매가/단가/링크/캡처/HTML 카드 확인 가능
  - X축 날짜 라벨 가독성 개선(기울임/중복 완화)
- 네이버 전용 확장 로직
  - 네이버 채널에서 전체 기간 등장 판매처를 폭넓게 노출하도록 API 설계

### 4.3 구현 완료 (운영/자동화)
- 스케줄 기반 자동 크롤링(배포 환경 Cron) 구성
- 수동 크롤링 실행 API 제공
- 카드 이미지 렌더/S3 업로드(옵션) 및 presigned URL 반환 지원

### 4.4 제한사항
- 네이버 API 원천 데이터 품질(상품명 표기 방식)에 따라 일부 수량 추정은 수동확인 필요
- 카드 렌더링/S3 기능은 런타임 의존성 및 환경변수 설정 충족 시 활성

## 6) 과제 결과 요약

- 메인 대시보드는 운영 관점 핵심 기능(현황 확인, 필터링, 수동검증, 크롤링 트리거, 증빙 확인)까지 구현 완료 상태이다.
- 네이버 채널은 데이터 수집-정제-DB적재-채널/셀러/타임라인 조회까지 End-to-End 흐름이 구현되어 실사용 가능한 단계이다.
- 현재 단계는 "MVP 완료 + 운영 안정화/고도화 진행 가능" 상태로 판단된다.

## 8) 최근 변경 반영 사항 (2026-03-26~2026-03-27 기준)

### 8.1 메인 대시보드
- 채널별 판매가 차트 구조 개편
  - 기존 채널 통합 5라인 -> `판매처 x 채널` 최대 10라인(네이버/쿠팡 분리)
  - 라인 토글 UI를 네이버/쿠팡 그룹으로 분리하여 가독성 개선
  - 라벨을 판매처명 위주로 단순화
- 데이터 기준 정합성 강화
  - 메인 차트를 판매처 개별 타임라인(`mall/timeline`) 기반으로 재구성
  - 주요/기타 카드 지표도 타임라인 기준으로 통일(현재 단가, 7일 변동폭, 기준가 이하 횟수)
- 기준가 이하 테이블 운영 기능 강화
  - 행 체크박스/전체선택/선택 삭제 버튼 추가
  - 삭제 즉시 재조회하여 화면 동기화
- 단가 인지성 개선
  - 기준가 이하/근접 구간을 색상(빨강/주황)으로 표시

### 8.2 네이버 채널
- 주요 셀러 운영 정책 확정
  - 네이버 주요 셀러 5개 고정 및 카드 순서 고정
  - `닥터다이어리(닥다몰)`과 `닥터다이어리(무화당)`은 분리 관리
  - 주요 셀러 그래프에는 `닥터다이어리(닥다몰)`만 포함
- 채널 추이/세부 추이 일관성 개선
  - 채널 판매가 추이를 개별 판매처 타임라인 기반으로 합성
  - 개별 판매처 페이지와 채널 추이 간 기준 불일치 해소
- 기타 네이버 판매처 지표 일원화
  - 기타 판매처 카드도 주요 셀러와 동일한 타임라인 계산 기준 적용

### 8.3 API 및 데이터 처리
- 수동확인 저장 API 연계 완료
  - `POST /products/manual-confirm`으로 수량 확정 시 단가 재계산 및 유효성 상태 갱신
- 일괄 삭제 API 추가
  - `POST /products/delete`로 선택 행 DB 삭제 지원
- 채널 필터 보정
  - `channel=naver` 조회 시 `channel`이 NULL/빈값인 과거 데이터도 포함
- trends 조회 정합성 보정
  - 표준화된 판매처명 기준으로 `tracked-malls/trends` 조회하도록 수정
  - 네이버/쿠팡 채널 조회 시 주요 5개 판매처를 조회 대상에 강제 포함

### 8.4 UI/UX 세부 개선
- 판매처 세부 그래프 X축 라벨 가독성 개선
  - 날짜 라벨 각도 조정 및 중복 완화
  - 일별 날짜 전체 표시 정책 반영
- 토글 UI 역할 분리
  - 그래프 라인 토글과 "주요 셀러 보기" 이동 버튼의 시각 스타일 분리

### 8.5 현재 운영 기준(요약)
- 메인 대시보드 차트: `판매처 x 채널` 분리(최대 10라인), 라인별 토글 가능
- 네이버/쿠팡 주요 셀러 페이지 차트: 고정 5개 주요 셀러 기준
- 카드/그래프/세부 타임라인 지표 계산 기준: 타임라인 기반으로 통일
